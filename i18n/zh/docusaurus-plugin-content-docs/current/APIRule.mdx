# API规则

### 关于API

为了在支付安全的前提下给商家带来简单、一致、易用的开发体验，我们推出的支付API，具有以下功能：

- 遵循统一的REST设计风格。
- 使用JSON作为数据交互的格式。
- 使用基于非对称密钥的SHA256-RSA数字签名算法，更有效地确保数据安全。

### SDK方式接入

我们提供API的官方SDK或Demo（目前包括Java、PHP、C#三种语言版本）。此外，我们还提供Postman调试工具、密钥生成工具、签名验证工具，您可以通过我们的GitHub访问。

我们建议商家基于官方提供的SDK开发应用程序。SDK封装了商户技术人员的请求签名和响应验证，简化了商户系统的开发。

### 自编码接入

在规则描述中，您将了解API的基本约定，如数据格式、参数兼容性、错误处理等。我们还将重点介绍API的身份验证机制（密钥/签名）。您可以遵循开发指南，使用命令行或您熟悉的编程语言逐步练习签名生成和签名验证。

如果您有任何问题，请随时向我们提问。
